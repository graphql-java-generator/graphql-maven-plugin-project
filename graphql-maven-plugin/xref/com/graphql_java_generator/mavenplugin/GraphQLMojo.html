20  /**
21   * <P>
22   * <B>This goal is <U>deprecated</U></B>. The <I>graphql</I> goal generates the java code from one or more GraphQL
23   * schemas. It allows to work in Java with graphQL, in a schema first approach.
24   * </P>
25   * It will be maintained in the future 2.x versions. The <I>generateClientCode</I> and <I>generateServerCode</I> should
26   * be used instead.<BR/>
27   * The <I>graphql</I> goal has two main modes:
28   * <UL>
29   * <LI><B>client mode:</B> it does the same jobs as the <I>generateClientCode</I> goal. It generates a class for each
30   * query, mutation and subscription type. These classes contain the methods to call the queries, mutations and
31   * subscriptions. That is: to execute a query against the GraphQL server, you just have to call one of this method. It
32   * also generates the POJOs from the GraphQL schema. The <B>GraphQL response is stored in these POJOs</B>, for an easy
33   * and standard use in Java.</LI>
34   * <LI><B>server mode:</B> it does the same jobs as the <I>generateServerCode</I> goal. It generates the whole heart of
35   * the GraphQL server. The developer has only to develop request to the data. That is the main method (in a jar project)
36   * or the main server (in a war project), and all the Spring wiring, based on graphql-java-spring, itself being build on
37   * top of graphql-java. It also generates the POJOs. An option allows to annotate them with the standard JPA
38   * annotations, to make it easy to link with a database. This goal generates the interfaces for the DataFetchersDelegate
39   * (often named providers) that the server needs to implement</LI>
40   * </UL>
41   * 
42   * @author etienne-sf
43   */
